# Requirements

## Context
- 项目：`hainan`（Java / Spring Boot）。
- 问题来源：前端“添加排班”调用 `/workPlan/saveAll` 失败，返回 `success=false, code=500, message=null`。
- 新增问题：使用“加载上次排班”后保存新日期时，旧日期排班被覆盖，导致旧日期数据丢失。
- 新增规则：排班加载需基于“用户当前选择日期”决定加载来源，而不是全库最大日期。

## Functional Requirements
- `/workPlan/saveAll` 在新增排班时，即使前端未传 `groupId` 也应可处理：
  - 通过 `classId` 反查班次所属考勤组并补齐 `groupId`。
  - 本地班次表未命中时，允许通过钉钉考勤组班次列表按 `classId` 反查 `groupId` 作为兜底。
- 保存前应进行基础字段校验：
  - `classId`、`userId`、`workDate`、`groupId` 均不能为空。
- 当人员需要跨考勤组调度时，后端应正确调用“移除旧组 + 加入新组”接口，且加组请求参数必须完整。
- 当参数不合法时，应返回明确错误信息，避免 `message=null`。
- 当“加载上次排班”得到的是旧日期记录（包含旧 `id`）时，保存到新日期必须新增记录，不允许覆盖旧日期原记录。
- 当未启用“加载上次排班”并编辑当前日期数据时，应继续按当前日期对应记录执行更新。
- 当“加载上次排班”=是 时，应加载**小于选择日期**且距离最近的一天排班记录。
- 当“加载上次排班”=否 时，应加载**选择日期当天**的排班记录；若无数据则返回空列表（前端不显示）。

## Non-Functional Requirements
- 不改变既有接口路径与入参格式，保持前端兼容。
- 失败信息可定位（包含缺失字段上下文），便于联调。
- 本地联调场景下，钉钉同步失败不应阻断本地排班落库（保证“添加排班/加载上次排班表”主流程可用）。
